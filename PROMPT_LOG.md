## Задание Средняя сложность №3: Написать Dockerfile для Rust-приложения.
**Инструмент:** Claude Sonnet 4.6 (claude.ai)
### Промпт 1
**Промпт:** У меня есть Rust приложение "Coffee Shop Visitors" (файлы загружены в чат). Напиши Dockerfile и .dockerignore.
Требования:
- Многоступенчатая сборка (builder + runner)
- Базовый образ: rust:1.75-slim (builder), debian:bookworm-slim (runner)
- Установка libsqlite3-dev в builder
- Установка libsqlite3-0 в runner
- Копирование Cargo.toml и исходного кода
- Оптимизация кэширования зависимостей (сначала Cargo.toml, потом код)
- Рабочая директория /app
- Переменная окружения DATABASE_URL=/app/data/coffee_shop.db (чтобы БД хранилась в отдельной папке)
- Открыть порт 3000
- Логи в stdout/stderr
- HEALTHCHECK: проверять GET /visitors каждые 30 секунд (таймаут 3с, start period 5с, retries 3). 
- .dockerignore должен исключать target/, .git, .env, tests/
Покажи готовые файлы.
**Результат:** Выведено 2 верных файла в соответствии с требованиями
### Промпт 2
**Промпт:** Проведи ревью всех файлов, включая новые
**Результат:** Найдена проблема в main.rs: В Dockerfile задан ENV DATABASE_URL=/app/data/coffee_shop.db, но main.rs хардкодит путь "coffee_shop.db" и переменную не читает. БД будет создаваться в /app/coffee_shop.db, а не в /app/data/ — volume-mount не будет работать.
Все исправлено, остальное корректно.
### Итого
- Количество промптов: 2
- Что пришлось исправлять вручную: созданы коммиты, проверка работы приложения без контейнера, код скопирован и перенесен из claude.ai. Собран Docker образ, проверка запуска контейнера.
- Время: ~60 мин
